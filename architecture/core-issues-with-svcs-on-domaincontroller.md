Documentation: Hyper-V Packer Builds on Domain Controllers
1. The Problem: "The Administrative Paradox"When an account (like ansible_svc) is elevated to high-privilege groups (e.g., Domain Admins or Hyper-V Administrators) on a Domain Controller, Windows applies Remote UAC restrictions.
Symptom: You can SSH into the box, but running hardware-level commands like Get-NetAdapter returns Access Denied (0x80041003).Cause: Windows strips the "Administrative Token" from network-based logons to prevent lateral movement by attackers.
2. Critical Fixes & Solutions
A. Punching through Remote UACTo allow ansible_svc to use its full administrative powers over an SSH connection, you must disable the remote token filter.
Action: Add the LocalAccountTokenFilterPolicy registry key.Command: ```powershellNew-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" -Name "LocalAccountTokenFilterPolicy" -Value 1 -PropertyType DWord -Force
B. Bypassing WMI Permission HellPacker attempts to auto-detect the host's IP address using PowerShell. On a DC, this query often hits a WMI security wall.The Problem: Syntax errors in win11.pkr.hcl when using host_ip_address (usually due to plugin version mismatches).The Solution: Use an Environment Variable in your Ansible playbook to force the IP, which tells Packer to skip the failing WMI query entirely.Implementation:YAML- name: Run Packer build
  win_shell: packer build win11.pkr.hcl
  environment:
    PACKER_HYPERV_HOST_IP: "192.168.4.61"
C. OpenSSH Host Key ConflictsFrequent permission changes and service restarts on the R430 can cause the SSH host key to appear "changed" to the Linux masternode, triggering MITM warnings.Solution: Flush the known hosts on the Linux side: ssh-keygen -R <IP_OR_HOSTNAME>.3. Required Group Memberships (The "Sweet Spot")On a DC, "Local Groups" don't exist. You must use Active Directory groups. For ansible_svc to successfully run Packer/Hyper-V, it must be in:Group NamePurposeHyper-V AdministratorsAllows creation/management of VMs.Remote Management UsersAllows login via WinRM/SSH.Performance Log UsersGrants read access to hardware/network stats.Domain Admins(Optional/Last Resort) Grants total oversight if WMI namespaces are locked.4. Best Practices for the HomelabIndentation is King: In Ansible playbooks, vars_files must never be indented under vars:. It must sit at the root of the play to ensure secrets are loaded before tasks run.Use Explicit Logon Formats: Always use the NetBIOS format DOMAIN\user (escaped as DOMAIN\\user in YAML) to ensure Windows doesn't try to authenticate against a non-existent local database.Pre-Flight Testing: Before running a 30-minute Packer build, verify permissions with a quick Ansible ad-hoc command:Bashansible wsdc-homelab -m win_shell -a "Get-NetAdapter" -k
Static IP for the Host: Always hardcode or use environment variables for the Host IP when working on a DC.
